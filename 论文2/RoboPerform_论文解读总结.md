## 第 1 节：引言（Introduction）

### 1.1 核心问题

当前人形机器人缺乏**即兴表现能力**，被局限在预定义动作或简单指令中。传统管线存在三大系统性问题：

1. **级联误差累积**：音频 → 人体动作生成 → 重定向 → 机器人跟踪，每一步引入误差
2. **推理延迟高**：多阶段串行处理
3. **声学-驱动映射松散**：高层音频线索与低层关节驱动之间联系薄弱

### 1.2 核心洞察：Motion = Content + Style

| 分量 | 含义 | 来源 | 作用 |
|------|------|------|------|
| **Content**（内容） | 机器人在"做什么" | 文本-to-运动模型编码为运动潜变量 | 指定核心任务（如"跳舞"还是"演讲"） |
| **Style**（风格） | "怎么"做 | 音频信号（音乐节拍、语音韵律） | 调制节奏、力度、时机等表现细节 |

### 1.3 框架概览：RoboPerform（Teacher-Student 架构）

| 组件 | 角色 | 核心设计 |
|------|------|---------|
| **ΔMoE 教师策略** | 在仿真中用 RL 学习高质量动作 | 残差混合专家架构 |
| **InfoNCE 音频-运动对齐** | 将音频特征映射到运动语义空间 | 对比学习 |
| **扩散学生策略** | 实时部署，端到端音频→动作 | Content/Style 解耦的扩散模型 |

## 第 2 节：相关工作（Related Work）

相关工作的三段递进逻辑：**为什么用运动跟踪 → 运动跟踪发展到什么程度 → 还缺什么（本文补上）**

### 2.1 Humanoid Whole-body Control（全身控制）

- **传统模型控制**：精确动力学建模 [5, 38] → 建模复杂，难以泛化
- **学习型方法**：手工设计任务奖励（走路 [41]、跳跃 [33]、跌倒恢复 [11] 等）→ 奖励工程繁琐，难以生成类人运动
- **分解控制** [19, 48]：拆分独立子策略 → 破坏身体协调性
- **层次框架** [39]：高层规划+低层执行 → 结构复杂

> **范式转变**：运动跟踪（motion tracking）——以人类运动数据为参考信号，统一训练目标，无需为每项技能单独设计奖励。

### 2.2 Humanoid Motion Tracking（运动跟踪）

| 阶段 | 代表工作 | 特点 |
|------|---------|------|
| 单运动模仿 | DeepMimic [32], ASAP [9], KungfuBot [43] | 高精度，但每次只能学一个动作 |
| 多运动统一策略 | OmniH2O [8], ExBody2 [16], GMT [4], KungfuBot2 [7] | 通用性强，但**无人用音频驱动** |

### 2.3 Modality-driven Locomotion（模态驱动运动）

| 工作 | 模态 | 局限 |
|------|------|------|
| LangWBC [36] | 语言 → 运动 | 无法处理复杂分布和未见指令 |
| RLPF [45] | LLM + 物理反馈 | 灾难性遗忘风险 |
| RoboGhost [23] | 语言潜变量驱动，免重定向 | 仅用语言 |
| **RoboPerform（本文）** | **音频驱动，免重定向** | **首个用音频作为隐式控制模态** |

## 第 3 节：方法（Method）

### 3.1 Overview（总览）

框架包含三大组件，依次串联：

```
① 音频-运动对齐（InfoNCE 对比学习）
   → 让音频潜变量携带运动学先验，无需训练专门的"音频→动作"生成器
        ↓
② ΔMoE 教师策略（PPO 强化学习训练）
   → 在仿真中以完整条件（含参考运动）学习高质量动作
        ↓
③ 扩散学生策略（DAgger 蒸馏）
   → Content（运动潜变量）定框架 + Style（对齐后的音频潜变量）调节奏
   → 端到端从音频直接生成机器人动作，低延迟、免重定向
```

### 3.2 ΔMoE（残差混合专家架构）

#### 3.2.1 MoE 是什么

**MoE（Mixture of Experts，混合专家）**：将一个大模型拆分成多个小专家网络，配上门控网络动态决定各专家的贡献权重。

```
输入 x ──→ 门控网络 ──→ 权重 [w₁, w₂, w₃, w₄]
                              │
        ┌─────┬─────┬─────┬────┤
        ↓     ↓     ↓     ↓
     Expert₁ Expert₂ Expert₃ Expert₄
        ↓     ↓     ↓     ↓
        a₁    a₂    a₃    a₄
        └─────┴─────┴─────┴──→ a = w₁a₁ + w₂a₂ + w₃a₃ + w₄a₄
```

**Vanilla MoE 的问题**：每个专家看到完整条件输入，各自输出完整动作，各专家输出特征严重重叠（t-SNE 可视化证实）→ 信息冗余，没有真正专业化。

#### 3.2.2 ΔMoE 的核心设计：残差增量学习

**关键思想**：不让专家独立建模完整动作，而是让每个专家只负责"新增条件带来的增量"。

**第一步：嵌套条件子空间划分**

条件抽象为 3 维向量 $\mathbf{c} = [c_1, c_2, c_3]^T$，构建嵌套子空间：

$$\{0\} = S_1 \subset S_2 \subset S_3 \subset S_4 = \mathbb{R}^3$$

| 专家 | 条件子空间 | 输入 | 建模含义 |
|------|-----------|------|---------|
| $e_1$ | $S_1 = \{0,0,0\}$ | 全零 | 无条件先验 $p(a)$，最基础的安全动作 |
| $e_2$ | $S_2 = \{c_1,0,0\}$ | 仅第一维条件 | 在 $e_1$ 基础上增加 $c_1$ 的信息 |
| $e_3$ | $S_3 = \{c_1,c_2,0\}$ | 前两维条件 | 在 $e_2$ 基础上增加 $c_2$ 的信息 |
| $e_4$ | $S_4 = \{c_1,c_2,c_3\}$ | 完整条件 | 在 $e_3$ 基础上增加 $c_3$ 的信息 |

**第二步：残差融合公式**

门控网络输出权重 $\mathbf{w} = [w_1, w_2, w_3, w_4]^T$（$\sum w_k = 1$），最终动作：

$$\boxed{a = w_1 a_1 + \sum_{i=2}^{4} w_i (a_i - a_{i-1})}$$

等价于加权条件增量之和：

$$a = \sum_{i=1}^{4} w_i \Delta a_i, \quad \Delta a_i = \mathbb{E}[a \mid c_{S_i}] - \mathbb{E}[a \mid c_{S_{i-1}}]$$

其中 $\Delta a_i = a_i - a_{i-1}$ 表示"新增第 $i$ 维条件后，动作该调整多少"。

**理解方式**：每个专家输出的是完整动作，但**贡献给最终结果的是它与上一级专家的差值**。这就保证了四个专家的贡献互不重叠。

**第三步：与 CFG 的理论联系**

ΔMoE 是 **Classifier-Free Guidance (CFG) 在连续多维条件空间下的结构化推广**：

| | CFG | ΔMoE |
|---|-----|------|
| 核心操作 | 有条件输出 − 无条件输出 | $a_i - a_{i-1}$ |
| 含义 | "条件带来的引导方向" | "新增条件维度的信息增益" |
| 层级 | 二元（有 vs 无） | 多层嵌套（$S_1 \subset S_2 \subset S_3 \subset S_4$） |

### 3.3 音频-运动对齐（Audio-Motion Alignment）

#### 3.3.1 目的是什么

让音频潜变量 $l_{audio}$ 携带**运动学信息**（kinematic priors），从而可以直接作为策略的条件输入，无需训练专门的"音频→运动"生成器。

#### 3.3.2 实现方式

- **Adaptor**：一个 6 层 Transformer，加入时序注意力来捕捉音频中的节奏结构
- **Motion Latent**：从预训练的 VAE 中提取
- **优化目标**：InfoNCE 损失（一种对比学习损失函数）

#### 3.3.3 InfoNCE 公式详解

$$\mathcal{L}_{\text{InfoNCE}} = -\frac{1}{N}\sum_{i=1}^{N} \log \frac{\exp\left(\text{sim}(l^{(i)}_{audio}, l^{(i)}_{motion})\right)}{\sum_{j=1}^{N} \exp\left(\text{sim}(l^{(i)}_{audio}, l^{(j)}_{motion})\right)}$$

其中 $\text{sim}(u, v) = \frac{u^\top v}{\tau}$，即两个归一化向量的缩放余弦相似度，$\tau$ 为温度超参数。

**拆解公式的逻辑**：

| 步骤 | 操作 | 含义 |
|------|------|------|
| ① | 准备一个 batch 的 $N$ 对音频-运动样本 | 每一对 $(i,i)$ 是匹配的正样本 |
| ② | 对每个音频 $i$，计算它与**所有 $N$ 个运动**的相似度 | $\text{sim}(l^{(i)}_{audio}, l^{(j)}_{motion})$ |
| ③ | 用 softmax 将相似度转化为"该运动被选中的概率" | $\frac{\exp(\text{sim}_{i,i})}{\sum_j \exp(\text{sim}_{i,j})}$ |
| ④ | 取 $-\log$ 作为损失 | $P\to 1$ 时损失 $\to 0$；$P \to 0$ 时损失 $\to$ 大 |

**本质**：一段鼓点音频 🥁，与它对应的那段舞蹈 💃 在向量空间中要**拉近**（正样本），与其他不相关的运动 🚶 要**推远**（负样本）。训练完成后，$l_{audio}$ 就被嵌入到了运动语义空间中，策略看到它就能"感受到"运动含义。

#### 3.3.4 关键纠正：InfoNCE 不是"先训练音频→动作生成器"

这篇文章的做法是**对比学习对齐**，而不是传统的"先训练一个生成器把音频转成动作，再让策略去跟踪"。对齐之后，$l_{audio}$ 直接作为策略的 style 条件输入，端到端生成机器人动作。这消除了中间环节的级联误差。

### 3.4 音频条件的策略蒸馏（Audio-conditioned Policy Distillation）

#### 3.4.1 核心思想：Motion = Content + Style

```
Content（运动潜变量）  →  恒定不变，来自文本 "一个人在跳舞/演讲"
                         →  决定去噪的"大方向"
                         
Style（对齐后的音频潜变量）→  随时间逐帧变化，来自音乐节拍/语音韵律
                         →  注入到扩散模型的每一层，调制节奏表现
```

#### 3.4.2 扩散模型注入方式

学生在去噪的每一层同时接收两种条件：

$$o_i = \text{Layer}_i\left(o_{i-1}, l_{motion}\right) + \alpha \cdot l_{audio}$$

- $l_{motion}$：内容条件（恒定），通过 **AdaLN** 注入
- $\alpha \cdot l_{audio}$：风格条件（逐帧变化），**加法式注入**到每层输出

#### 3.4.3 训练方式：DAgger 蒸馏

1. 在仿真中用学生策略 rollout，采集访问过的状态
2. 在这些状态上向教师策略（ΔMoE）查询最优动作 $\hat{a}$
3. 用 MSE 损失训练扩散学生：$\mathcal{L} = \|a - \hat{a}_t\|^2$
4. 学生使用 $x_0$-prediction 参数化（直接预测干净动作，而非预测噪声）
5. 推理时使用 **DDIM-2**（仅 2 步去噪），延迟仅 5.3ms

## 附录：关键概念澄清

### A. CFG（Classifier-Free Guidance）≠ 分类器引导

> ⚠️ **常见混淆点**：CFG 和 Classifier Guidance 是两种不同的引导方法。

| | Classifier Guidance（分类器引导） | CFG（无分类器引导） |
|------|------------------------------|-------------------|
| 提出者 | Dhariwal & Nichol (2021) | Ho & Salimans (2022) |
| 是否需要额外模型 | **需要**：额外训练一个能处理噪声图的分类器 | **不需要**：同一个扩散模型同时学会两种模式 |
| 怎么做 | 分类器对噪声中间态 $x_t$ 分类，用其梯度引导 | 训练时以概率（~15%）随机丢弃条件，让模型同时学会 $p(x\mid c)$ 和 $p(x)$ |
| 推理公式 | $\nabla \log p(x_t) + \lambda \nabla \log p_\phi(c\mid x_t)$ | $\text{无条件输出} + \gamma \times (\text{条件输出} - \text{无条件输出})$ |
| 缺点 | 需额外训练分类器；分类器能力限制引导质量 | 训练时需随机丢弃条件（增加训练成本） |

**CFG 的训练技巧**：同一个模型，训练时以约 15% 概率把条件替换为空 token $\varnothing$。这样模型同时学会：
- 看到条件 $c$ 时输出 $p(x \mid c)$（条件模式）
- 看到空条件 $\varnothing$ 时输出 $p(x)$（无条件模式）

推理时对两个预测做外推：无条件输出 + $\gamma$ × (条件输出 − 无条件输出)，$\gamma$ 越大，生成结果越贴合条件。

### B. 潜变量（Latent Variable）

就是**编码之后的紧凑向量表示**。原始音频波形（44.1kHz × 10秒 = 441,000 个数值）被压缩为 256 维向量；原始运动序列（23 关节 × 30fps × 10 秒 = 6,900 个数值）被压缩为 64 维向量。压缩后只保留核心语义信息，且维度相近才能做对比学习对齐。

### C. 这篇论文中 RL 的角色

PPO 是训练教师策略的核心算法（Actor-Critic 框架，含 GAE、clip 机制、域随机化、课程学习），**但 PPO 算法本身不是本文的创新**。本文的创新在于：
- **策略网络架构**：ΔMoE（神经网络结构设计）
- **多模态对齐**：InfoNCE 音频-运动对齐（表征学习/对比学习）
- **训练范式**：Teacher-Student DAgger 蒸馏（模仿学习 + 扩散模型）
- **奖励工程**：20 项精细跟踪 + 正则化奖励（RL 应用实践）

这代表了当前机器人 RL 论文的典型模式：**RL 是发动机（不改），创新在车身设计（策略结构、表征学习、训练技巧）**。
